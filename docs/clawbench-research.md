## 二、核心架构

### 2.1 零适配透传（Zero-Adaptation Passthrough）

ClawBench 的架构哲学是**不重新实现 AI 能力**，而是将现有 AI 编程智能体 CLI 作为后端引擎，通过 Go Web 服务封装为 HTTP API + SSE 流式接口。前端仅负责渲染和交互。

```
┌─────────────────┐
│  移动端 / 浏览器  │
│  (Vue 3 + TS)    │
└───────┬─────────┘
        │ HTTP / SSE
┌───────▼─────────┐
│  Go Web Server   │
│  (net/http)      │
└───────┬─────────┘
        │ exec / stdio
┌───────▼─────────┐
│  AI CLI 后端      │
│  (Claude Code,   │
│   Gemini CLI...) │
└─────────────────┘
```

源码层面，`internal/ai/cli_backend.go` 通过 `exec.CommandContext` 调用 CLI 工具，并使用 `bufio.Scanner` 进行流式输出。此外还有一个 ACP（Agent Client Protocol）后端，使用长连接 stdio 作为替代传输方式。

这种设计保留了底层 CLI 工具的所有原生能力，包括：
- 工具调用（Tool Calling）
- 深度思考（Deep Thinking）
- Skill 系统
- MCP 协议

### 2.2 技术栈

| 层级 | 技术 |
|------|------|
| 后端 | Go 1.25+，net/http，SQLite（pure-Go modernc.org/sqlite） |
| 前端 | Vue 3 + Vite + TypeScript |
| 终端 | PTY + WebSocket + xterm.js |
| 向量存储 | DuckDB（用于 RAG） |
| 部署 | 绿色单文件部署，一个二进制 + 静态资源，零运行时依赖 |

### 2.3 客户端-服务端架构

采用 C/S 架构，Go Web 服务 + SSE 流式通信。前端静态资源通过 `go:embed` 嵌入二进制中，实现真正的单文件部署。

## 四、核心功能

### 4.1 智能体运行平台

ClawBench 将自身定位为**完整的智能体运行平台**，而非仅仅是一个「聊天壳」：

- **Agent 数据库存储**：智能体配置持久化到 SQLite
- **自动发现**：启动时扫描已安装的 AI CLI，自动注册
- **Cron 定时执行**：支持定时任务调度
- **多 Agent 调度**：不同任务匹配不同智能体

### 4.2 设置向导

- 支持 **23 个 WizardReady LLM 供应商**（包括 OpenAI、Anthropic、DeepSeek 等）
- 内置 **567 个支持工具调用的模型**
- 供应商模型数据通过 Python 脚本从 models.dev API 自动生成，via `go:embed` 嵌入二进制
- API Key 使用 **AES-256-GCM** 加密存储，修改登录密码时自动轮换加密密钥

> 注：23 个供应商和 567 个模型的数字截至 2026-05-31 生成日期，会随数据更新而变化。另有 5 个企业级供应商未标记为 WizardReady。

### 4.3 TTS 语音合成系统

ClawBench 集成了完善的 TTS 系统，AI 回复会**自动总结后朗读**：

| 引擎 | 类型 | 特点 |
|------|------|------|
| Edge TTS | 免费云端 | 默认引擎 |
| MiniMax | 付费云端 | 最高质量 |
| Piper | 本地离线 | 快速 |
| Kokoro | 本地离线 | 高质量中文 |
| MOSS-TTS-Nano | 本地离线 | 多语言，48kHz，声音克隆，0.1B 参数 |

总结后端支持 **12+ 种**（包括 Claude CLI、Gemini CLI、CodeBuddy CLI、Codex CLI、DeepSeek TUI 以及 OpenAI/Anthropic 格式的远程 API）。TTS 语音和定时任务共用同一套总结配置。

### 4.4 Web 终端

基于 **PTY + WebSocket + xterm.js** 架构：

- 每个新连接创建独立的 PTY 会话
- 唯一的 8 位十六进制会话 ID
- 独立环形缓冲区和空闲计时器
- 默认支持每个项目最多 **10 个并发 PTY 会话**（可配置）

## 五、设计哲学

### 5.1 「AI 负责改，我负责看」

ClawBench **刻意不提供直接的文件编辑能力**。所有文件修改必须通过 AI Agent 完成，项目重点打造的是：

- Markdown 和代码的**预览体验**
- 预览过程中与 AI 的**内联交互**（选中文本即可向 AI 提问或请求修改）

> **注意**：用户仍可通过内置 Web Terminal 使用 vim/nano 编辑文件，因此这是一个**设计理念**而非硬性技术限制。

### 5.2 零适配透传

不造轮子，不重新实现 AI 能力，而是充当现有 CLI 工具与移动端之间的桥梁。这保证了：
- 底层工具的功能不被削减
- 新功能跟随上游自动获得
- 维护成本低

## 十、总结

ClawBench 是一个**有明确定位和独特设计理念的项目**。它通过「零适配透传」架构，巧妙地将 9+ 个现有 AI 编程 CLI 工具统一到一个移动端友好的 Web 界面中，而不是重新发明轮子。单文件部署、自动发现、TTS 语音合成、Cron 调度等功能使其超越了简单的聊天壳，成为一个相对完整的智能体运行平台。

**优势**：
- 架构设计简洁务实，维护成本低
- 多后端支持，用户可按需切换
- 移动端优先的设计填补了市场空白
- 单文件部署极大简化了运维

**风险**：
- 社区规模小，项目持续性待观察
- 安全文档不充分
- 强依赖上游 CLI 工具的稳定性和兼容性
- 第一方声明中存在被验证驳回的营销性表述
